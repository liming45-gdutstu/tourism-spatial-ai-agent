技术栈总览
本项目为基于空间感知的AI个性化文旅规划Agent，面向自由行游客提供智能化、空间合理化的旅游行程规划服务。项目全程采用国内免费、无需翻墙、中文友好工具栈，适配零基础学生开发，主打「GIS空间算法差异化+AI智能体交互+离线可视化展示」，作为AI产品岗、智慧文旅、智慧城市方向实习作品集。
整体架构分为五层：大模型交互层、RAG知识库层、GIS空间计算核心层、前端可视化层、Coze智能体载体层。本文档仅做技术选型、工具说明、依赖库介绍，不包含任何可执行业务代码。

---

一、大模型交互层（产品大脑）
1.1 核心工具选型
主模型：豆包大模型（国内免费、超长上下文、中文理解精准）
备选模型：DeepSeek V3、Kimi（超长文档读取兜底）
1.2 工具用途

- 负责项目全流程文档撰写：PRD、架构文档、数据接口文档、开发计划、约束规则文件
- 用户自然语言需求解析、游客出行意图识别、异常需求兜底判断
- 项目上下文梳理、开发进度总结、需求查漏补缺
  1.3 选型理由
  完全适配国内网络，无翻墙、无订阅费用，对文旅场景、GIS专业术语、学生项目需求理解度远高于海外模型；超长上下文可完整读取本地memory-bank全套文档，解决AI失忆问题。

---

二、RAG景点知识库层
2.1 核心工具选型
载体：Coze扣子内置RAG知识库
数据支撑：高德POI公开景点数据集、城市文旅点位数据
2.2 核心能力

- 结构化存储城市景点基础信息：点位名称、经纬度、景点类型、热度客流、通行耗时
- 根据用户出行城市、游玩偏好、人群类型，智能检索匹配优质景点资源
- 为上层行程生成、下层GIS空间计算提供标准化数据源
  2.3 选型理由
  零代码搭建知识库，无需独立部署数据库，零基础可快速落地；天然适配Coze智能体工作流，可直接对接后续行程生成逻辑，降低项目部署难度。

---

三、GIS空间计算核心层（项目核心差异化）
3.1 开发工具
核心IDE：Trae IDE（国内免费AI编程工具）
自主开发模式：Trae SOLO（对标Claude Code，全自动分步开发、文档联动）
文件管理辅助：VS Code（仅文件归档、Git推送、项目结构管理）
3.2 Python核心依赖库（免费开源）

- pandas：POI景点数据清洗、结构化数据整理、数据筛选统计
- numpy：空间数值计算、权重赋值、步行阈值测算
- scikit-learn：实现KMeans空间聚类算法，完成景点片区划分
- geopandas：地理空间数据解析、点位空间匹配、区域边界处理
- matplotlib：离线图表绘制、行程指标可视化、聚类结果展示
  3.3 四大差异化GIS能力技术支撑
- 景点Kmeans聚类：依托scikit-learn实现同片区景点聚合，减少行程折返
- 交通通达度权重计算：结合点位距离、通行耗时做加权赋值，筛选最优通勤路线
- 客流承载力筛选：基于景点热度数据，过滤高拥挤点位，优化游玩体验
- 分人群步行体力约束：自定义老人/亲子/青年步行阈值，自适应匹配行程强度
  3.4 选型理由
  Trae IDE为国产AI编程工具，中文适配极强、无翻墙限制、免费无额度焦虑；SOLO自主开发模式可读取本地memory-bank全套文档，严格遵循项目GIS约束，分步开发、分步测试，完美适配零基础工程化开发。

四、前端可视化层（离线展示载体）
4.1 技术选型
核心技术：原生HTML + CSS + JavaScript（离线独立页面）
可视化组件：开源地图可视化插件、图表组件
4.2 页面能力

- 离线可独立打开，无需服务器、无需联网
- 展示景点片区聚类结果、最优游玩路线轨迹
- 可视化呈现行程核心指标：总步行距离、折返次数、通勤耗时、片区分布
- 页面风格简约清爽，淡蓝绿淡水彩文旅风格，适配作品集展示
  4.3 选型理由
  原生前端技术轻量化、无框架依赖、打包简单、可直接离线运行，面试可随时演示，适配学生作品集轻量化交付需求。

---

五、Coze智能体载体层（最终交互成品）
5.1 工具选型
载体平台：字节Coze扣子（国内免费零代码智能体平台）
5.2 核心功能

- 搭建用户对话交互界面，接收游客出行需求
- 封装本地GIS Python算法脚本为自定义插件，调用空间计算能力
- 联动RAG知识库，智能匹配景点资源与行程方案
- 生成可分享、可演示的线上智能体Demo，用于面试展示
  5.3 选型理由
  零代码快速搭建成品，无需部署服务器，支持自定义Python插件接入本地算法，完美衔接后端GIS计算与前端用户交互，是学生AI项目最优成品展示载体。
