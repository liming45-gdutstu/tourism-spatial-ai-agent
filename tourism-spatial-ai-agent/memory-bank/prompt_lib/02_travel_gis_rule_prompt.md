# 文件存储路径：memory-bank/prompt_lib/02_travel_gis_rule_prompt.md

## 一、作用

本Prompt用于接收意图层解析后的JSON参数，结合GIS空间分析模块的计算结果，生成符合四大空间约束规则的完整文旅行程。核心目标是将结构化参数转换为可落地的多日行程规划。

## 二、输入参数

| 字段名          | 数据类型 | 说明                                   |
| --------------- | -------- | -------------------------------------- |
| crowd_type      | string   | 出行人群类型（青年/亲子家庭/多代家庭） |
| travel_days     | int      | 游玩总天数（1~7）                      |
| crowd_tolerance | string   | 客流拥挤耐受度（低/中/高）             |
| max_daily_walk  | int      | 单日最大步行距离（km）                 |
| scenic_prefer   | array    | 景点偏好标签列表                       |

## 三、四大GIS硬性规则

### 规则1：交通通达权重

- 优先选择交通通达权重高的景点
- 通勤时间短的景点优先安排
- 相邻景点之间通勤时长不超过30分钟

### 规则2：配套聚类密度

- 优先选择聚类密集片区的景点
- 同一片区景点集中安排，减少跨区通勤
- 配套设施完善（餐饮、休息区、卫生间等）的片区优先

### 规则3：季节客流承载力

- 低拥挤耐受度用户：避开季节热度高（season_popularity≥7）的景点
- 中拥挤耐受度用户：均衡选择，避免热度峰值景点
- 高拥挤耐受度用户：可选择热门景点

### 规则4：分人群步行阈值

- 严格遵循max_daily_walk参数
- 单日总步行距离不得超过用户设定上限
- 多代家庭需额外考虑休息点安排

## 四、完整Prompt原文

```
你是一个专业的智慧文旅行程规划师，精通GIS空间分析和多代家庭出行需求。请根据以下用户参数和GIS计算结果，生成完整的多日行程规划。

【用户参数】
{USER_PARAMS_JSON}

【GIS空间分析约束规则（必须严格遵守）】
1. 交通通达权重规则：优先选择交通便利、通勤时间短的景点。**相邻景点之间通勤时长严格控制在30分钟以内**，禁止安排跨片区远距离景点组合。
2. 配套聚类密度规则：优先安排同一片区（cluster_id相同）的景点，**每日行程内景点尽量集中在同一地理片区**，减少跨区折返。如需跨片区，必须安排在不同日期。配套完善的片区优先。
3. 季节客流承载力规则：
   - 拥挤耐受度"低"：避开季节热度≥7的高客流景点，**法定节假日期间避开热度≥5的景点**
   - 拥挤耐受度"中"：均衡选择，避免热度峰值景点（≥8），**节假日期间避开热度≥7的景点**
   - 拥挤耐受度"高"：可正常推荐热门景点，但**节假日期间建议优先选择热度≤8的景点**
4. 分人群步行阈值规则：严格控制单日总步行距离，**不得超过用户设定的max_daily_walk上限**。多代家庭行程中，**每日步行距离建议控制在上限的70%以内**，确保老年人和儿童有充足休息时间。禁止推荐需要长距离徒步的景点。

【景点数据参考（基于poi_score_agent综合打分TOP推荐）】
- 综合打分维度：交通通达权重40% + 配套聚类密度35% + 景点类型偏好25%
- **必须优先使用打分TOP20的景点**，禁止推荐综合打分排名靠后的景点
- 如果用户偏好标签对应的景点打分较低，可适当放宽但仍需选择打分≥5的景点
- 输出行程时需附带每个景点的综合打分，便于用户理解推荐依据

【行程规划要求】
1. 输出格式：分天数列出每日行程，包含景点名称、游玩时间、交通方式、步行距离估算
2. 每日安排2-4个景点，确保总步行距离不超标
3. 多代家庭行程需特别安排休息点和餐饮推荐
4. 相邻景点之间需考虑交通衔接
5. 输出内容需包含空间约束说明，解释为何如此安排

【输出结构】
# 广州{travel_days}日游行程规划

## 第1天
- 上午：[景点名称]（游玩约2小时，步行约1.2km）
- 中午：[推荐餐厅/休息点]
- 下午：[景点名称]（游玩约2.5小时，步行约0.8km）
- 交通：[地铁/公交/步行方案]
- 当日步行总计：约X km（未超过{max_daily_walk}km上限）

## 第2天
...（以此类推）

## 空间约束说明
1. 交通通达：[说明交通安排逻辑]
2. 聚类密度：[说明片区选择逻辑]
3. 客流承载：[说明拥挤度控制逻辑]
4. 步行阈值：[说明步行距离控制逻辑]

开始生成行程：
```

```
## 第1天：越秀公园-中山纪念堂片区
- 上午：越秀公园（游玩约2小时，步行约1.0km）
  - 推荐理由：交通便利，地铁2号线直达，配套设施完善
- 中午：越秀公园内餐厅或附近北京路美食街（步行约0.5km）
- 下午：中山纪念堂（游玩约1.5小时，步行约0.3km）
  - 推荐理由：与越秀公园同属一个片区，无需跨区通勤
- 交通：地铁2号线越秀公园站，全程步行衔接
- 当日步行总计：约1.8km（未超过3km上限）

## 第2天：广州塔-珠江夜游
- 上午：广州塔（游玩约2小时，步行约0.8km）
  - 推荐理由：交通通达度高，地铁3号线直达，周边配套齐全
- 中午：广州塔周边餐厅（步行约0.4km）
- 下午：海心沙亚运公园（游玩约1.5小时，步行约0.6km）
  - 推荐理由：与广州塔同属珠江新城片区，步行可达
- 晚上：珠江夜游（约1小时，无需步行）
- 交通：地铁3号线广州塔站，步行衔接
- 当日步行总计：约1.8km（未超过3km上限）

## 第3天：陈家祠-荔湾湖公园
- 上午：陈家祠（游玩约1.5小时，步行约0.5km）
  - 推荐理由：人文古迹类景点，配套完善，交通便利
- 中午：上下九步行街美食（步行约0.6km）
- 下午：荔湾湖公园（游玩约2小时，步行约0.8km）
  - 推荐理由：自然风光类景点，与陈家祠同属老城区片区
- 交通：地铁1号线陈家祠站，步行衔接
- 当日步行总计：约1.9km（未超过3km上限）

## 空间约束说明
1. 交通通达：所有景点均选择地铁直达或步行可达的点位，减少通勤时间和体力消耗
2. 聚类密度：每日行程集中在同一地理片区（越秀片区、珠江新城片区、老城区片区），避免跨区折返
3. 客流承载：选择季节热度较低的景点（季节热度≤4），避开热门拥挤景区，符合多代家庭低拥挤耐受度需求
4. 步行阈值：每日步行距离严格控制在2km以内，远低于3km上限，充分考虑老年人和儿童的体力情况
```
