## 三、迭代使用说明

### 1. 新增BadCase流程

当发现新的线上问题或用户反馈时，按照以下步骤操作：

1. **记录问题现象**：清晰描述大模型输出的错误行程问题，包含具体场景和影响
2. **定位根因**：分析问题产生的根本原因，判断是哪个分层Prompt的约束缺失或逻辑漏洞
3. **编写优化方案**：针对根因设计具体的Prompt修复方案，确保可直接替换进现有Prompt
4. **验证效果**：将优化后的Prompt投入测试，对比修复前后的输出效果，确认问题已解决
5. **更新文档**：将完整记录追加到本台账表格中，保持记录完整性

### 2. Prompt同步更新

每次在本台账中新增记录后，必须同步更新对应的分层Prompt文件：

- **Case1/C2/C3/C5** → 更新 `02_travel_gis_rule_prompt.md`（GIS文旅专属规则层）
- **Case4** → 更新 `01_intent_parse_prompt.md`（通用意图抽取层）
- 涉及异常场景 → 更新 `03_fallback_error_prompt.md`（异常兜底降级层）

### 3. 版本管理

建议在每次Prompt更新时记录版本号，便于追溯和回滚：

```
版本记录：
- v1.0（2026-07-08）：初始版本，覆盖5类高频BadCase
- v1.1（待定）：新增XX问题修复
```

在面试或作品集展示时，可重点讲解以下内容：
