# 多代家庭文旅GIS AI行程Agent

> 基于空间感知的AI个性化文旅规划智能体，为多代家庭提供智能化、空间合理化的旅游行程规划服务

---

## 📍 项目简介

本项目针对OTA行业「千人一面」的行程推荐痛点，创新引入GIS空间分析能力，构建面向多代家庭（青年、亲子、多代同堂）的分层个性化行程规划系统。通过Kmeans片区聚类、交通通达度权重、季节客流承载力、分人群步行阈值四大差异化能力，实现从「景点堆砌」到「空间最优」的行程规划升级。

**核心价值**：为不同人群（青年8km/亲子5km/多代家庭3km）提供差异化行程方案，解决传统行程推荐忽略体力差异、景点跨区通勤长、旺季拥挤体验差等行业痛点。

---

## 🎯 用户调研与痛点分析

### 用户调研结论

基于320+线上游记差评挖掘、5组三代同游用户深度访谈，发现三大核心矛盾：

| 痛点类型           | 占比 | 具体表现                       |
| ------------------ | ---- | ------------------------------ |
| 老人步行体力受限   | 41%  | 行程步行距离超标，老人无法完成 |
| 缺少儿童适配景点   | 28%  | 景点选择未考虑儿童兴趣与安全   |
| 行程跨片区折返严重 | 31%  | 景点分布分散，通勤时间过长     |

**用户手动规划行程平均耗时3-5小时**，现有工具无法平衡老中青三代诉求。调研样本聚焦珠三角2-5天短途家庭出游场景。

### 竞品短板分析

拆解携程、飞猪、美团三大OTA文旅AI工具：

- **均仅依靠大模型纯文本生成**，缺少GIS空间计算能力
- **无步行负荷校验**，无法识别体力约束
- **无多层级人群约束**，无法满足多代家庭差异化需求
- **路线脱离现实**，未考虑片区通勤与实际可达性
- **仅做通用出行工具**，未深耕多代家庭垂直场景

## ✨ 四大核心产品亮点

### 亮点1：GIS片区Kmeans聚类

- 基于景点经纬度自动划分游玩片区，减少跨区通勤
- 同片区景点优先串联，降低单日步行折返
- 聚类结果可视化展示，直观呈现片区分布
- **差异化壁垒**：区别于纯文本行程生成，具备空间量化能力

### 亮点2：多代人群分层参数表单

| 人群类型 | 步行上限 | 拥挤耐受度 | 景点偏好           |
| -------- | -------- | ---------- | ------------------ |
| 青年     | 8km      | 高         | 热门景点、网红打卡 |
| 亲子家庭 | 5km      | 中         | 亲子友好、自然风光 |
| 多代家庭 | 3km      | 低         | 低拥挤、人文古迹   |

**设计洞察**：通过320+用户调研，精准定义三代人差异化需求参数。

### 亮点3：采集-生成-确认-导出闭环流程

1. **需求采集**：自然语言解析用户出行意图，构建UserProfile用户画像
2. **行程生成**：GIS空间算法计算最优路线，量化步行负荷与通勤时间
3. **方案确认**：可视化卡片展示行程指标，支持参数二次调整
4. **数据导出**：支持CSV/SHP格式数据输出，导出行程攻略

### 亮点4：景点+5分钟本地美食配套

- 推荐景点周边5分钟步行范围内的特色美食
- 配套餐饮信息提升行程实用性
- 减少游客额外搜索成本

## 🏃 完整业务链路

### 前端用户操作流程

```
用户进入首页
    ↓
录入出行基础参数（目的地、天数、人群类型）
    ↓
前端校验拦截（空值/负数异常提示）
    ↓
配置步行、景点、预算偏好
    ↓
后端GIS算法计算（约15秒）
    ↓
展示片区地图 + 行程卡片
    ↓
支持修改偏好重生成 / 导出行程攻略
```

### 后端GIS算法计算流程

```
高德POI数据读取（187条广州文旅景点）
    ↓
Kmeans片区聚类 → 生成cluster_id
    ↓
步行负荷分配 → 按人群类型分配每日步行量
    ↓
预算/营业时间校验 → 100%拦截闭园景点
    ↓
交通通达度权重计算 → 通勤最优排序
    ↓
季节客流承载力筛选 → 过滤高拥挤景点
    ↓
分人群步行体力约束 → 确保不超标
    ↓
AI综合打分推荐 → TOP景点排序
    ↓
分层Prompt生成结构化行程
```

## 🎨 原型页面功能拆解

### 页面1：基础出行录入页

- 目的地输入框（支持城市选择）
- 游玩天数选择（1-7天）
- 交通方式选择（自驾/公共交通/步行）
- 人群构成：成人/老人/儿童人数及年龄段
- 前端实时异常提示（空值、负数、超出范围）
- 下一步按钮（参数校验通过方可点击）

### 页面2：游玩偏好配置页

- 步行耐受度滑块（低/中/高）
- 景点标签多选（自然风光、人文古迹、美食街区、网红打卡、博物馆）
- 预算档位选择（经济型/舒适型/豪华型）
- 返回上一页按钮
- 生成行程按钮

### 页面3：GIS行程总览页

- 色块片区地图（可视化展示景点分布）
- 横向景点卡片（分天展示，支持左右滑动）
- 同片区美食推荐（5分钟步行可达）
- 四大GIS量化指标面板：
  - 综合评分：0-10分
  - 交通权重：通勤便利度
  - 配套密度：片区景点丰富度
  - 季节拥挤度：当前季节客流情况
- 底部操作栏：修改参数、确认行程、导出攻略

### 页面4：景点详情弹窗

- 景点大图展示
- 步行适配标签（老人友好/亲子推荐/青年首选）
- 5分钟步行可达本地美食列表
- 出行避坑提示（闭园时间、拥挤时段、注意事项）
- 右上角关闭按钮

### 弹窗功能

| 弹窗类型     | 触发场景          | 功能说明               |
| ------------ | ----------------- | ---------------------- |
| 参数缺失引导 | 用户输入不完整    | 友好提示补充必要信息   |
| 加载状态     | 行程计算中        | 显示进度，预计等待时间 |
| 异常报错兜底 | 接口超时/数据为空 | 给出备选推荐，引导重试 |
| 分享/导出    | 用户主动操作      | 支持分享链接和导出攻略 |

## 🧪 测试用例设计

### 测试用例1：多代家庭（核心场景）

- **用户输入**：带爸妈和孩子去广州玩3天，不想走太多路，喜欢看风景和历史古迹
- **预期结果**：每日步行≤2.5km，推荐低拥挤景点（越秀公园、中山纪念堂），配套休息点多

### 测试用例2：亲子家庭

- **用户输入**：一家三口暑假去广州玩2天，喜欢游乐园和自然风光
- **预期结果**：每日步行≤4km，推荐亲子友好景点（广州塔、海心沙），均衡拥挤度

### 测试用例3：青年

- **用户输入**：两个年轻人国庆去广州玩5天，不怕人多，喜欢美食和拍照打卡
- **预期结果**：每日步行可达7km，推荐热门景点（北京路、长隆）

### 测试用例4：参数缺失（异常场景）

- **用户输入**：我想去广州玩
- **预期结果**：友好引导补充信息（游玩天数、人群类型），推荐热门景点

### 测试用例5：无匹配景点（边界场景）

- **用户输入**：带老人去广州玩1天，步行不超过1km，只要博物馆
- **预期结果**：返回少量匹配景点，提示放宽约束
